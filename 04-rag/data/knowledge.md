# AI 与大语言模型核心知识库

## 什么是大语言模型（LLM）

大语言模型（Large Language Model，简称 LLM）是一种基于深度学习的自然语言处理模型，通过在海量文本数据上进行预训练，学习语言的统计规律和语义表示。LLM 的核心架构是 Transformer，由 Google 在 2017 年的论文《Attention is All You Need》中提出。

现代 LLM 的代表包括 OpenAI 的 GPT 系列、Anthropic 的 Claude 系列、Google 的 Gemini 系列，以及国内的 DeepSeek、通义千问等。这些模型的参数量从数十亿到数千亿不等，具备强大的文本理解和生成能力。

LLM 的工作原理是基于"下一个 Token 预测"（Next Token Prediction）。给定一段文本前缀，模型会预测最可能的下一个 Token，并通过自回归的方式逐步生成完整的回复。Token 是文本的最小处理单位，中文通常 1-2 个字对应一个 Token。

## Prompt Engineering（提示工程）

Prompt Engineering 是指通过精心设计输入提示（Prompt）来引导大语言模型生成期望输出的技术。好的 Prompt 能显著提升模型回答的质量、准确性和相关性。

### 核心技巧

**System Prompt**：系统提示是设置在对话开头的指令，用于定义模型的角色、行为边界和输出格式。例如："你是一位资深的 Python 开发者，请用简洁专业的语言回答技术问题。"

**Few-shot Learning**：通过在 Prompt 中提供几个示例（输入-输出对），帮助模型理解任务模式。研究表明，3-5 个高质量示例通常能显著提升任务表现。

**Chain-of-Thought（CoT）思维链**：引导模型逐步推理而非直接给出答案。在 Prompt 中加入"让我们一步一步思考"（Let's think step by step）等引导语，可以显著提升模型在数学推理和逻辑分析任务上的准确率。

**结构化输出**：通过在 Prompt 中指定输出格式（如 JSON Schema），可以让模型返回结构化数据，便于程序解析和后续处理。

## RAG（检索增强生成）

RAG（Retrieval-Augmented Generation）是一种结合信息检索和文本生成的技术架构。它通过在生成回答前先从外部知识库中检索相关信息，将检索结果作为上下文注入到 Prompt 中，从而增强模型的回答质量。

### RAG 的核心优势

1. **减少幻觉**：LLM 有时会"编造"看似合理但实际错误的信息（即幻觉现象）。RAG 通过提供真实的参考资料，大幅降低幻觉发生的概率。

2. **知识时效性**：LLM 的知识截止于训练数据的时间点。RAG 可以接入实时更新的知识库，让模型获取最新信息。

3. **领域专业化**：通过构建特定领域的知识库（如企业内部文档、专业技术手册），RAG 可以让通用模型具备领域专家的能力。

4. **可追溯性**：RAG 的回答可以附带来源引用，用户可以验证信息的准确性。

### RAG 的完整流程

RAG 的工作流程分为两个阶段：

**离线阶段（Indexing）**：
- 文档加载：读取各种格式的文档（PDF、Markdown、HTML 等）
- 文本分块（Chunking）：将长文档切分成适当大小的文本块
- 向量化（Embedding）：使用 Embedding 模型将文本块转换为向量表示
- 存储索引：将向量存入向量数据库（如 ChromaDB、Pinecone、Qdrant）

**在线阶段（Retrieval & Generation）**：
- 查询向量化：将用户问题转换为向量
- 相似度检索：在向量数据库中找到最相似的文本块
- 上下文构建：将检索结果组织成上下文
- 增强生成：将上下文和问题一起发送给 LLM 生成回答

## 文本分块（Chunking）策略

文本分块是 RAG 中的关键步骤。分块质量直接影响检索效果和最终回答质量。

**固定大小分块**：按固定字符数切分文本，通常设置 overlap（重叠）来避免语义被截断。优点是实现简单，缺点是可能在语义中间切断。

**递归字符分块**：按分隔符优先级递归切分。例如先按段落（\n\n）切，如果块太大再按句子切，最后按字符切。这种方式能较好地保留语义完整性。

**语义分块**：利用 Embedding 模型判断相邻句子的语义相似度，在语义变化较大的位置进行切分。效果最好但计算成本较高。

推荐的分块大小通常在 200-1000 个 Token 之间，overlap 设置为块大小的 10%-20%。具体参数需要根据文档特点和检索效果进行调优。

## Embedding（向量嵌入）

Embedding 是将文本转换为固定维度数值向量的技术。语义相似的文本在向量空间中的距离更近，这是向量检索的基础。

常用的 Embedding 模型包括：
- **OpenAI text-embedding-3-small**：1536 维，API 调用，效果好但需付费
- **all-MiniLM-L6-v2**：384 维，开源模型，可本地运行，适合学习和小规模应用
- **BGE 系列**：中文效果优秀的开源模型

Embedding 的质量直接决定检索的准确性。选择 Embedding 模型时需要考虑：维度大小、语言支持（中英文）、推理速度、是否支持本地部署等因素。

## 向量数据库

向量数据库是专门用于存储和检索高维向量的数据库系统。与传统数据库的精确匹配不同，向量数据库支持近似最近邻（ANN）搜索，能够高效找到与查询向量最相似的结果。

主流向量数据库包括：
- **ChromaDB**：轻量级，纯 Python/JS 客户端，适合开发和学习
- **Pinecone**：云托管服务，开箱即用，适合生产环境
- **Qdrant**：高性能，支持丰富的过滤条件
- **Milvus**：分布式架构，适合大规模场景

相似度计算方法包括余弦相似度（Cosine Similarity）、欧氏距离（Euclidean Distance）和内积（Dot Product）。其中余弦相似度最常用，它衡量两个向量方向的一致性，值域为 [-1, 1]，值越大表示越相似。
