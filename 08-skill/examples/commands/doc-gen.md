## Documentation Standards

1. **Module Overview** — Purpose, architecture, dependencies
2. **API Reference** — Every exported function/class/type with:
   - Description
   - Parameters (name, type, description, default)
   - Return value
   - Usage example
3. **Configuration** — Environment variables, config files
4. **Examples** — Common use cases with runnable code

## Output Format

Use JSDoc/TSDoc for inline documentation. For standalone docs, use Markdown with:
- Table of contents
- Code blocks with syntax highlighting
- Chinese descriptions
